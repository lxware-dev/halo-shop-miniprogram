# 写在前面

在开始使用本项目之前，建议先阅读本文，了解项目定位、使用前提与文档结构，避免在错误的方向上花费时间。

## 项目定位

本仓库是 **Halo 商业版配套的微信小程序前端**，不包含独立后端，需连接已部署并可正常访问的 Halo 服务。

## 使用前提

- 本项目依赖 Halo 侧的**商品、订单、支付、用户**等商业能力，需使用 **Halo 商业版**才能完整对接上线。
- 确保当前小程序商户符合[交易类小程序运营规范](https://developers.weixin.qq.com/miniprogram/product/jiaoyilei/yunyingguifan.html)。
- 对于实物类小程序，还需要接入[订单发货管理](https://developers.weixin.qq.com/miniprogram/product/jiaoyilei/fahuoguanligongneng.html)。

## 下一步

如果你希望最终上线一个微信小程序，建议从「本地准备」开始，依次完成后台数据配置、静态配置与上线准备，最后进入「发布上线」完成提审与正式发布。

**1. 本地准备**（[prepare-local.md](./prepare-local.md)）：在本地机器上将项目跑起来，完成小程序侧的品牌化与接口配置。

**2. 后台数据**（[prepare-backend.md](./prepare-backend.md)）：在 Halo 管理端完成商品、分类、首页、登录、支付等数据录入，确保各核心流程可以与前端联调跑通。

**3. 配置参考**（[config.md](./config.md)）：查看登录方式、法务链接、帮助中心、客服入口等配置字段，按你的业务需求完成小程序静态配置。

**4. 上线准备**（[prepare-go-live.md](./prepare-go-live.md)）：侧重首次上传体验版、提交审核或正式发布前，在**生产环境**与**微信公众平台**侧备齐所有条件，并完成支付验收、协议页检查与真机验证，避免因域名、HTTPS、合规等问题导致真机或审核阶段失败。

**5. 发布上线**（[release.md](./release.md)）：当你已经完成体验版验证，并确认生产环境、支付、协议页与公众平台配置都没有问题后，再进入这一阶段，按顺序完成**上传版本、提交审核、审核通过后正式发布**。
