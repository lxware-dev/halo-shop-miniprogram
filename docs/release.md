# 发布上线

本文说明从**构建上传**到**提交审核、正式发布**的完整流程。平台规则以 [微信公众平台](https://mp.weixin.qq.com) 最新说明为准。

## 上传前检查

> [!IMPORTANT]
> 上传前请先完成 [prepare-go-live.md](./prepare-go-live.md) 中的所有准备工作。以下为最低核对项：

- [ ] 已完成生产构建：`pnpm build:mp-weixin`（见 [deployment.md](./deployment.md)）。
- [ ] `halo.baseURL`、登录方式、协议链接等均为**生产环境**配置。
- [ ] 已在**真机**上验证核心路径：浏览商品 → 登录 → 下单 → 支付。
- [ ] 小程序基本信息（名称、简介、类目、头像）已在公众平台填写完整，且与资质一致。

## 版本管理

在 `src/manifest.json` 中维护合理的版本号：

```json
{
  "versionName": "1.0.0",
  "versionCode": "100"
}
```

> [!TIP]
> 建议在微信开发者工具上传时的**版本备注**中写清功能变更与依赖的后端版本，便于后续回溯。

## 上传步骤

1. 使用**微信开发者工具**打开生产构建目录（通常为 `dist/build/mp-weixin`）。
2. 点击工具栏的**上传**按钮，填写版本号与备注。
3. 登录 [微信公众平台](https://mp.weixin.qq.com)，进入**版本管理**，找到刚上传的版本。
4. 点击**提交审核**，按提示填写测试路径、测试账号（如需要）与补充说明。
5. 审核通过后，在版本管理中选择**发布**，完成正式上线。

## 提审材料建议

- 若登录非开放注册，提供可**复现核心流程**的测试账号。
- 涉及支付的，在备注中说明为真实商户测试环境，或按微信要求提供说明材料。
- 若类目需要资质，提前在公众平台完成资质上传与审核。

## 常见审核关注点

> [!NOTE]
> 以下为历史常见驳回原因，非穷尽列表，以微信官方审核标准为准。

- 服务类目与小程序实际功能不符。
- 存在**未说明的用户数据收集**，或与隐私政策内容不一致。
- 存在**测试数据、占位文案、无效链接**。
- 核心功能无法在无特殊条件下完成（需提供清晰的测试路径与账号）。

## 发布后运维

**监控**

- 结合 Halo 日志、微信开发者工具的**运维中心**与客服反馈监控异常。

**版本回滚**

- 可在微信公众平台将线上版本回退到上一个正式版（以平台当前能力为准）。

**热修复**

- 小程序代码修复需重新**上传、提审**（除非微信提供紧急修复机制）。
- 对于重要缺陷，建议尽快构建新版本、写明变更说明后提交审核。
