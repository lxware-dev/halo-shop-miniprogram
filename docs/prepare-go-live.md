# 上线前准备

本文说明在**首次上传体验版、提交审核或正式发布**之前，需要在生产环境与微信公众平台侧完成的所有准备工作。

本地开发与联调见 [prepare-local.md](./prepare-local.md)；上传与提审操作见 [release.md](./release.md)。

## 适用时机

- 已完成本地/测试环境联调，准备切换到**生产级 HTTPS 域名**。
- 准备在微信开发者工具中**上传**构建产物，或提交**审核**。

## 第一步：生产环境配置

在正式构建前，确认以下配置均已指向**生产环境**：

- [ ] `src/config/app.config.json` 或 `app.config.local.json`（也可通过 CI 注入）中的 `halo.baseURL` 指向正式 Halo 服务，不再指向仅内网可访问的地址。
- [ ] `src/manifest.json` 中的版本号、小程序名称等与产品/运营约定一致。
- [ ] `.env` 中 `VITE_MOCK_ENABLED` 为 `false`，确保生产构建不走 Mock 数据。

> [!TIP]
> 构建命令与产物路径详见 [deployment.md](./deployment.md)。

## 第二步：微信公众平台配置

登录 [微信公众平台](https://mp.weixin.qq.com)，完成以下配置。具体入口以平台当前界面为准。

### 服务器域名（必须配置）

前往 **开发 → 开发管理 → 开发设置 → 服务器域名**，配置以下域名：

| 类型             | 说明                                                                 |
| ---------------- | -------------------------------------------------------------------- |
| **request**      | Halo API 域名，需与 `halo.baseURL` 的**协议 + 主机名**一致，不含路径 |
| **uploadFile**   | 若业务涉及文件上传，按实际接口域名配置                               |
| **downloadFile** | 商品图、Logo 等外链图片的域名，需满足微信配置要求                    |

> [!IMPORTANT]
> 所有合法域名均须使用 **HTTPS**，且域名需已完成 ICP 备案。

### 业务域名（按需配置）

若使用 `<web-view>` 组件打开用户协议、隐私政策等 H5 页面，需在**业务域名**中配置对应域名，并完成校验文件的部署。

### 用户隐私与合规

- 按微信要求填写**用户隐私保护指引**，采集的数据项需与隐私政策内容保持一致。
- 小程序的**类目、名称、简介、头像**需与实际功能和资质一致。

### 接口权限（按需申请）

如收货地址、用户手机号等敏感权限，在实际调用前需在后台申请并确保已通过审核。

## 第三步：真机验证

> [!WARNING]
> 微信开发者工具可临时关闭合法域名校验，但**真机预览、体验版、正式版**均受合法域名限制。请务必在真机上完成以下验证。

- [ ] 使用**体验版**在真机上完整走通：浏览商品 → 登录 → 加入购物车 → 下单 → 支付。
- [ ] 确认 Halo 使用**有效的 HTTPS 证书**（证书链不完整会导致真机请求失败）。
- [ ] 确认 `business.legalDocuments` 中的协议链接均可正常访问。

## 上线前检查清单

完成以上步骤后，逐项确认：

**配置与网络**

- [ ] 生产 `halo.baseURL` 已确认，且与 **request 合法域名**完全一致。
- [ ] 已在微信公众平台完成**服务器域名**配置。
- [ ] 商品图等外链资源的域名已配置 **downloadFile** 合法域名。

**安全**

- [ ] 无测试账号口令硬编码在代码中。
- [ ] 含敏感信息的配置文件（如 `app.config.local.json`）未提交到公开仓库。

**合规**

- [ ] 登录、支付主路径已在**真机**上完整验证（见 [payments.md](./payments.md)）。
- [ ] `business.legalDocuments` 等链接均为 **HTTPS** 且可正常访问。
- [ ] 用户隐私保护指引已在微信公众平台填写完整。
