# 上线前准备

本文说明在**首次上传体验版、提交审核或正式发布**之前，需要在生产环境与微信公众平台侧完成的所有准备工作，包括域名、合规、支付验收与真机验证。

## 适用时机

- 已完成本地/测试环境联调，准备切换到**生产级 HTTPS 域名**。
- 准备在微信开发者工具中**上传**构建产物，或提交**审核**。

## 第一步：生产环境配置

在正式构建前，先将 `app.config.local.json` 中的 `halo.baseURL` 切到正式 Halo 服务，不要再指向仅内网可访问的地址；接着同步确认 `src/manifest.json` 中的版本号、小程序名称等信息已经和产品、运营约定一致，最后再检查 `.env` 中 `VITE_MOCK_ENABLED` 为 `false`，避免生产构建仍然走 Mock 数据。

> [!TIP]
> 构建命令与产物路径详见 [deployment.md](./deployment.md)。

## 第二步：微信公众平台配置

登录 [微信公众平台](https://mp.weixin.qq.com)，完成以下配置。具体入口以平台当前界面为准。

### 服务器域名（必须配置）

前往 **开发 → 开发管理 → 开发设置 → 服务器域名**，配置以下域名：

| 类型             | 说明                                                                 |
| ---------------- | -------------------------------------------------------------------- |
| **request**      | Halo API 域名，需与 `halo.baseURL` 的**协议 + 主机名**一致，不含路径 |
| **uploadFile**   | 若业务涉及文件上传，按实际接口域名配置                               |
| **downloadFile** | 商品图、Logo 等外链图片的域名，需满足微信配置要求                    |

> [!IMPORTANT]
> 所有合法域名均须使用 **HTTPS**，且域名需已完成 ICP 备案。

### 业务域名（按需配置）

若使用 `<web-view>` 组件打开用户协议、隐私政策等 H5 页面，需在**业务域名**中配置对应域名，并完成校验文件的部署。

### 用户隐私与合规

- 按微信要求填写**用户隐私保护指引**，采集的数据项需与隐私政策内容保持一致。
- 小程序的**类目、名称、简介、头像**需与实际功能和资质一致。

### 接口权限（按需申请）

如收货地址、用户手机号等敏感权限，在实际调用前需在后台申请并确保已通过审核。

## 第三步：支付与订单验收

### 支付联调检查清单

1. 先按 [prepare-backend.md](./prepare-backend.md) 完成 Halo 后台中的**微信小程序支付**配置，包括 AppID、商户号、商户私钥、证书序列号、API V3 密钥等。
2. 确认微信小程序 AppID 与微信商户号已在微信支付商户平台完成**绑定关联**。
3. 确认生产环境 `halo.baseURL` 已配置为 **HTTPS** 合法域名，并已加入微信公众平台的 `request` 合法域名。
4. 使用**体验版 + 真机**跑通一次下单、调起支付、支付完成、订单状态更新的完整流程。

### 常见支付失败原因

| 现象                   | 可能原因                                           |
| ---------------------- | -------------------------------------------------- |
| 无法调起支付控件       | 订单未创建成功、服务端未返回支付参数、商户配置错误 |
| 提示权限或配置错误     | AppID / 商户号不匹配、小程序未关联商户             |
| 支付成功但订单状态未变 | 异步通知失败、Halo 回调 URL 配置错误或被防火墙拦截 |

> [!TIP]
> 以上问题需结合 **Halo 后台配置**、**服务端日志** 与 **微信商户平台** 一起排查。前端可记录请求失败时的错误提示与订单号，便于定位问题。

### 订单与物流

- 订单列表、详情、确认收货、物流追踪等功能依赖 Halo 返回的数据结构，后台需正确维护订单状态与物流单号。
- 若商品为虚拟商品或服务类，需在后台与业务规则中明确标记**不需要物流**，避免前端展示异常。

## 第四步：真机验证

> [!WARNING]
> 微信开发者工具可临时关闭合法域名校验，但**真机预览、体验版、正式版**均受合法域名限制。

先使用**体验版**在真机上完整走通一遍浏览商品、登录、加入购物车、下单、支付的流程；如果过程中出现请求失败，再回头确认 Halo 使用的是**有效的 HTTPS 证书**，避免因为证书链不完整导致真机访问失败。最后再打开 `business.legalDocuments` 中配置的协议链接，确认它们都能正常访问。

## 下一步

到这里，你应该继续回看三件事：生产 `halo.baseURL` 是否已经和微信公众平台中的 `request` 合法域名完全一致，商品图等外链资源是否已经配置好 `downloadFile` 合法域名，以及用户隐私保护指引是否已经在公众平台填写完整。

如果还准备提交审核，再补充确认代码里没有硬编码的测试账号口令，带敏感信息的本地配置文件（如 `app.config.local.json`）也没有被提交到公开仓库。完成这些后，就可以继续阅读 [release.md](./release.md) 开始上传和提审。
